# Prompt Dataset Builder

Android-приложение для ручного создания текстового датасета по изображениям из `civita.red` или совместимого Civitai API.

## Формат одной позиции

Каждая запись содержит ровно три строки:

```json
{
  "command": "Составь подробный англоязычный промпт...",
  "input": "Описание изображения, введённое пользователем",
  "output": "Исходный или отредактированный промпт"
}
```

Служебные ID нужны только приложению и не попадают в экспорт.

## Как проходит разметка

1. Приложение показывает изображение.
2. Исходный промпт скрыт.
3. Пользователь вводит собственное описание в поле `input`.
4. После ввода минимум пяти символов становится доступна кнопка раскрытия.
5. Приложение показывает исходный промпт в поле `output`.
6. Пользователь при необходимости исправляет его и сохраняет запись.
7. ID изображения отмечается обработанным и больше не появляется в ленте.

## Что происходит с изображениями

- файл не добавляется в датасет;
- постоянное сохранение изображений не используется;
- дисковый кэш Coil отключён;
- HTTP-кэш клиента отключён;
- изображение остаётся только в оперативной памяти на время просмотра;
- локально сохраняется только ID источника как отметка обработанной картинки.

## Возможности MVP 0.1.0

- лента изображений с доступными промптами;
- настраиваемый API endpoint;
- пользовательский API-ключ;
- NSFW-режим только с API-ключом и подтверждением возраста 18+;
- API-ключ хранится локально через Android Keystore и AES-GCM;
- локальная SQLite-база;
- исходный промпт скрыт до ввода своего описания;
- редактирование `output` перед сохранением;
- временный пропуск;
- постоянное исключение неподходящей картинки;
- просмотр и удаление записей;
- экспорт JSONL только с `command`, `input`, `output`;
- GitHub Actions для сборки debug APK и release APK/AAB.

## Сборка APK на GitHub

1. Создайте пустой репозиторий GitHub.
2. Загрузите в него всё содержимое папки проекта.
3. Откройте вкладку **Actions**.
4. Выберите workflow **Android CI**.
5. Нажмите **Run workflow**.
6. После завершения откройте запуск.
7. В разделе **Artifacts** скачайте `prompt-dataset-debug-apk`.
8. Распакуйте архив и установите APK на Android.

Workflow также запускается автоматически после commit в ветку `main` или `master`.

GitHub Actions самостоятельно устанавливает JDK 17 и Gradle 8.13, затем выполняет:

```text
lintDebug
testDebugUnitTest
assembleDebug
```

## Подписанный release

Для подписанного APK/AAB добавьте в GitHub Secrets:

- `ANDROID_KEYSTORE_BASE64`
- `KEYSTORE_PASSWORD`
- `KEY_ALIAS`
- `KEY_PASSWORD`

После push тега вида `v0.1.0` workflow **Android Release** создаст GitHub Release и приложит APK, AAB и файл контрольных сумм.

Без секретов подписи workflow публикует только debug APK; неподписанные release-файлы пользователю не выдаются.

## Настройка API

По умолчанию используется:

```text
https://civita.red/api/v1/images
```

В настройках есть переключатель на официальный совместимый endpoint:

```text
https://civitai.com/api/v1/images
```

Endpoint должен возвращать JSON с массивом `items`. Для каждого элемента ожидаются:

- `id`;
- `url`;
- `width`;
- `height`;
- `meta.prompt`;
- опционально `nsfwLevel`;
- опционально `username`.

API-ключ передаётся только по HTTPS:

```text
Authorization: Bearer <API_KEY>
```

Для NSFW-режима требуется непустой пользовательский API-ключ, а запрос получает параметр:

```text
nsfw=X
```

Для безопасного режима:

```text
nsfw=None
```

## Формат экспорта

Файл `.jsonl`, одна позиция на строку:

```json
{"command":"...","input":"...","output":"..."}
```

ID изображения, внутренний статус и даты в экспорт не включаются.

## Требования

- Android 8.0+;
- minSdk 26;
- compileSdk/targetSdk 36;
- JDK 17;
- Gradle 8.13;
- Android Gradle Plugin 8.13.2.

## Ограничение источника

Публичный формат endpoint `civita.red` не удалось независимо подтвердить. Поэтому адрес API редактируется в настройках, а официальный Civitai endpoint добавлен как резервный вариант. Если `civita.red` возвращает другой JSON, менять потребуется только `CivitaApiClient`, база и интерфейс останутся прежними.
